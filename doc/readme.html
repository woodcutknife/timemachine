1 Introduction

backup is a file backup tool under Linux, it can be used to backup and restore data locally or remotely. It has those features:

  • Incremental backup.
  • Configuration with a tiny file.
  • Easy to set excluded files and folders.
  • Pretty display of backup status.
  • Can limit the number of backup versions.
  • Can restore data to a specified version.
  • Remote backup support - use SSH.

And this documentation is generated with the help of htlatex.

2 Installation

backup use autoconf to manage configuration and installation. So to install just enter:

$ aclocal
$ autoconf
$ automake --add-missing
$ ./configure

If there’s no error during configuration, then enter:

$ make install

The backup then has been installed to /usr/local/bin/backup, and the documentation has been installed to /usr/local/share/doc/backup/readme.html.

To uninstall, just enter:

$ make uninstall

Hint: perhaps you’ll need to sudo.

3 Configuration

The config file is defaultly located at /usr/local/etc/backup.conf, but you can modify this in /usr/local/bin/backup as you want.

3.1 BACKUP

To specify globally what files and folders you want to backup. Remember to insert a DELIMITER between two path.

3.2 DELIMITER

To split BACKUP.

3.3 TARGET

To specify where to place your backups. It could be a path on your remote server.

3.4 BACKUP_UPPER_LIMIT

To configure the upper limit of the number of versions backup should save.

3.5 SSH

To switch mode from backup locally and backup remotely. Remember that if you want to use ssh, it’s supposed to connect passwordlessly.

3.6 SSH_USER

To specify the ssh user name.

3.7 SSH_HOST

To specify the ssh host name(or ip).

3.8 DAILY_BACKUP_HOUR

To specify the hour of daily backup time.

3.9 DAILY_BACKUP_MINUTE

To specify the minute of daily backup time.

4 Usage

The general command line usage is:

backup {config,status,push,restore,daily,help}

4.1 backup config

This command simply tells where the config file locates.

4.2 backup status

This command is used to check backup status.

backup status [file-or-folder]

If a file or a folder name is specified, backup will check the backup for this file or folder to find all available versions. If not, backup will check status of all the files and folders specified in config file.

4.3 backup push

This command is used to backup data(push them!).

backup push [file-or-folder]

Just like the status command, if a file of a folder name is specified, backup will push this file or folder to your backups. If not, backup will push all the files and folders specified in config file.

4.3.1 .backupignore

Just like .gitignore, backup support user-defined .backupignore files.
Wherever you want to ignore something, just put the relative path in a file named .backupignore.

For example, now you’re at a folder named test, and you want to ignore test/file-to-ignore, just create .backupignore in this folder, add:

file-to-ignore

4.4 backup restore

This command is used to restore data to a version of the data.

backup restore file-or-folder version

There is an example:

backup restore ./test.sh 2014-05-25T15_13_18

4.5 backup daily

This command is used to start or stop daily backup. The backup time is configured in the config file.

backup daily {start,stop}

4.6 backup help

This command tells you how to use backup, though it’s a little bit ugly.

backup help {config,status,push,restore,daily,help}
